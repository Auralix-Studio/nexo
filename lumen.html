Qué es Lumen

Lumen es un asistente conversacional que vive dentro de Nexo. Acepta preguntas en lenguaje natural y construye respuestas a partir de dos fuentes:

  1. Datos del usuario disponibles localmente (perfil, horario, cuotas, calificaciones) obtenidos previamente por la aplicación desde SIGMA o la Intranet UPLA.
  2. Base de conocimiento estática sobre la universidad (carreras, sedes, trámites, asignaturas) distribuida con la aplicación.

La inferencia se ejecuta íntegramente en el procesador del dispositivo. No se realizan solicitudes a servicios de IA externos durante el uso del asistente.

Cómo funciona

Cuando el usuario formula una consulta, el flujo de procesamiento es el siguiente:

1

Clasificación

Un router liviano analiza la consulta con expresiones regulares para identificar las categorías relevantes (horario, pagos, calificaciones, carreras, trámites, información institucional).

2

Construcción del prompt

Se ensambla en memoria un prompt que incluye únicamente los bloques de datos identificados como relevantes. Esto evita saturar el contexto del modelo y mantiene el rendimiento.

3

Inferencia

MediaPipe LLM Inference ejecuta el modelo sobre la CPU o GPU del dispositivo y emite la respuesta token a token.

4

Renderizado

La interfaz del chat muestra los tokens conforme se generan, con efecto de escritura progresiva. Al finalizar, el contexto se reinicia.

Variantes disponibles

Durante la activación del asistente, el usuario selecciona la variante que mejor se adapte a su dispositivo.

  • Velocidad30-50 tok/s

Modelo reducido optimizado para tiempos de respuesta breves.

Hardware moderno

Lumen Estándar

  • Tamaño~530 MB
  • RAM en ejecución~800 MB
  • Velocidad15-25 tok/s
  • RAM recomendada4 GB o superior

Mayor capacidad. Respuestas más elaboradas y mejor comprensión.

La selección se realiza desde Lumen → Configuración → "Cambiar modelo". La operación elimina la variante actual y descarga la nueva.

Capacidades

El asistente está diseñado para responder a consultas como:

  • "¿Cuál es mi próxima clase?"
  • "¿Cuánto debo este mes?"
  • "¿Cuál es mi promedio?"
  • "¿Cómo trámito una constancia de matrícula?"
  • "¿Qué carreras tiene la facultad de Ingeniería?"
  • "¿En qué sede está la biblioteca principal?"

También admite conversación casual y preguntas sobre el propio asistente o sobre Nexo.

Limitaciones

  • No accede a internet durante la inferencia.
  • No modifica información en SIGMA ni en otros sistemas. Es un agente de solo lectura.
  • No envía mensajes a profesores ni a terceros.
  • No conserva el historial entre consultas en esta versión. La persistencia se contempla para la versión 1.3.
  • No sustituye a la asesoría académica oficial. Para trámites importantes verificar la información en las fuentes institucionales.
  • No tiene la capacidad de razonamiento de los modelos comerciales en la nube. Los modelos utilizados son significativamente más pequeños.

Privacidad técnica

La afirmación de que Lumen opera completamente en el dispositivo se sustenta en los siguientes hechos verificables:

  1. El archivo del modelo se descarga una única vez desde un release público de GitHub. Esta es la única solicitud de red que origina el módulo Lumen.
  2. La inferencia se ejecuta mediante MediaPipe LLM Inference, una biblioteca C++ desarrollada por Google que opera sobre TensorFlow Lite. Ambas son bibliotecas locales estándar.
  3. La consulta del usuario y la respuesta generada residen exclusivamente en la memoria del proceso y se descartan al finalizar el chat.

El código del módulo Lumen puede revisarse bajo acuerdo de confidencialidad para verificar estas afirmaciones de forma independiente.
